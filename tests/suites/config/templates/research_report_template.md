# 调研报告模板

<!--
【模板是什么】
调研报告是调研 Agent（research_agent）在完成调研任务后输出的结构化结果文档，
记录了调研目标、方法、关键发现、方案对比、建议方案以及参考来源。

【模板的作用】
1. 结果结构化 — 按统一格式组织调研结果，便于上层 Agent 解析和使用
2. 过程可追溯 — 记录调研方法和参考来源，支持结果验证
3. 方案可对比 — 提供多维度方案对比，辅助决策
4. 建议可执行 — 给出带优先级和理由的明确建议

【如何使用本模板】
1. 调研 Agent 通过 static_vars 加载本模板内容
2. 按模板章节顺序填写调研结果
3. [必填] 章节必须填写，[可选] 章节根据调研需要填写
4. 最终输出到 docs/{调研主题}_research_report.md（具体路径以验收标准为准）

【适用场景】
| 调研类型 | 说明 |
|----------|------|
| technology | 技术调研，重点在方案对比和选型建议 |
| resource | 资源调研，重点在现有资源清单和复用建议 |
| market | 市场调研，重点在竞品分析和市场趋势 |
| solution | 方案调研，重点在解决方案和最佳实践 |
| general | 通用调研，综合信息整理 |

【与其他模板的关系】
本模板为独立调研报告模板，由 research_agent 使用。
调研结果可被上层 Agent（如 planner）引用，作为任务规划的输入依据。
-->
