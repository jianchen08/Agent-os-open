# Agent OS — 项目愿景

> **文档性质**：项目总纲，定义 Agent OS 是什么、为什么存在、要走向哪里。
> **读者**：项目所有参与者（开发者、设计师、产品决策者）。
> **本文讲"为什么"，具体版本规划与时间线见 [ROADMAP.md](../ROADMAP.md)。

***

## 一、一句话定义

**Agent OS 是你的全能 AI 助理 —— 既是生产力引擎，也是个人伙伴，更是可以自定义人格与形象的数字生命。**

它不是一个聊天机器人，不是一个工具箱，不是一个 Agent 框架。
它是一个**以你为中心的 AI 操作系统**，能思考、能执行、能进化、能陪伴。

***

## 二、与同类项目的差异

市面上已有不少 Agent 相关的项目，灵汐 AgentOS 的定位与之不同：

| 项目类型 | 代表 | 它们做什么 | AgentOS 的不同 |
|---------|------|-----------|---------------|
| **Agent 框架** | LangChain / LangGraph / AutoGen | 提供 SDK，让开发者**写代码**拼装 Agent | 配置即产品——YAML 定义 Agent，**不写代码**也能组装 |
| **Agent 应用** | AutoGPT / BabyAGI | 单一目标的自主任务执行 | 全能 + 可陪伴——从生产力到情感到创意，人设可定制 |
| **低代码平台** | Dify / Coze / FastGPT | 可视化搭建聊天应用 / 工作流 | 管道可观测可干预、自进化闭环、多通道接入、开放协议 |
| **聊天产品** | ChatGPT / Claude | 通用对话助手 | 以你为中心、有记忆有人设、能长出界面、可自托管 |

**一句话区分**：框架要你写代码，应用只做一件事，低代码做聊天机器人；**AgentOS 是一个可自托管、可进化、有人设的 AI 操作系统**——你配置它，它替你干活、陪你交流、还越用越懂你。

***

## 三、非目标（我们不做什么）

明确边界，避免误解和无效贡献：

- **❌ 不做通用聊天机器人** —— 不和 ChatGPT 竞争"无所不知的问答"。我们追求的是**能执行、能交付、能陪伴**的专属助理，不是百科问答。
- **❌ 不训练基础模型** —— 不做预训练或从头训练 LLM。我们消费模型能力（OpenAI / Anthropic / DeepSeek / GLM / Ollama），把精力放在**编排、工具、记忆、进化**上。
- **❌ 不绑定单一云厂商或模型** —— 核心不依赖任何特定 LLM 厂商；多模型路由是基础能力，换模型不改配置结构。
- **❌ 不做闭源 SaaS** —— 项目以 Apache 2.0 开源，可完全自托管；云端托管是未来可选，不是唯一形态。
- **❌ 不追求"万能内核"** —— 内核保持最小（管道引擎 + 配置 + 插件加载），**能力靠插件长出来**，不为每个场景往内核塞功能。

> 如果你的需求落在上述"不做"的范围，AgentOS 可能不是合适的工具——这没问题，我们宁可把一件事做透。

## 四、产品定位

### 4.1 全能助理

Agent OS 不局限于某个领域。它同时是：

| 角色        | 它能做什么                      | 用户感受            |
| --------- | -------------------------- | --------------- |
| **生产力引擎** | 写代码、管项目、分析数据、编排创意生产、自动化工作流 | 工作效率翻倍，复杂任务一键完成 |
| **个人助理**  | 日程管理、信息整理、知识管理、生活规划、决策辅助   | 生活中多了一个得力管家     |
| **创意伙伴**  | 写小说、做设计、生成音乐、编排剧本、游戏开发     | 灵感无限，创意闭环       |
| **学习教练**  | 知识检索、学习路径规划、答疑解惑、技能训练      | 随时随地有私人导师       |
| **数字伙伴**  | 自定义人格、角色扮演、情感陪伴、虚拟形象       | 它不只是工具，更是一个"人"  |

**核心主张**：你不需要在不同 APP 之间切换，Agent OS 是你唯一的 AI 入口。

### 4.2 超高自由度与千人千面

**超高自由度是千人千面的前提**。Agent OS 把"定义 Agent"的自由度彻底交给用户——人格、风格、知识、行为、工具集、约束、记忆全部可配置，不写一行代码就能组装出截然不同的 Agent。正因为每个人都能自由定义，才有了"千人千面"：一千个用户，一千种 Agent。

- **说话风格**：正式商务、轻松幽默、温柔体贴、毒舌吐槽……由你决定
- **知识领域**：专业模式侧重技术，伙伴模式侧重生活，可以混合搭配
- **行为模式**：主动还是被动，简洁还是详细，直接还是委婉
- **记忆**：记住你的偏好、习惯、过往对话，越用越懂你

## 五、三维愿景

Agent OS 的愿景由三个维度构成，它们相互增强，形成飞轮效应：

```
        ┌──────────────────────────────────────┐
        │              飞轮循环                  │
        │                                       │
        │   ┌──────────┐                        │
        │   │ 全能闭环  │ 覆盖更多场景            │
        │   │ 创意审批  │ ──────────────┐        │
        │   └────┬─────┘               │        │
        │        │ 发现新需求           ▼        │
        │        ▼               ┌──────────┐   │
        │   ┌──────────┐         │ 超级终端  │   │
        │   │  自进化   │         │ 能力长出  │   │
        │   │  越用越强  │         │   界面    │   │
        │   └────┬─────┘         └────┬─────┘   │
        │        │ 生成新能力           │         │
        │        └─────────────────────┘         │
        │              用户体验提升               │
        │                ↓ 回到全能闭环            │
        └──────────────────────────────────────┘
```

### 5.1 全能闭环 —— 创意与生产的完整闭环

从创意构思到最终交付，形成完整的闭环——AI 生产制品 → 人类审批 → 批注反馈 → AI 修改 → 再审批 → 通过交付。

每一个产出都经过"生成-审批-反馈-迭代"的闭环，确保质量可控、方向可调。

**关键能力**：

- AI 自主执行具体工作（写作、编程、设计、建模……）
- 人类只在关键节点做方向性审批
- 批注、打回、修改、版本对比——完整的审批工作流
- 适用于小说创作、软件开发、影视制作、游戏开发等所有创意生产场景

### 5.2 自进化 —— 越用越强

Agent 不仅是被执行的系统，更是一个可以自我进化的系统。它通过两条回路持续变强：

- **复盘回路**：每条管道结束后，系统对执行过程做深度回顾，把经验沉淀到知识库；下一次遇到类似任务，这些经验会被检索注入，让 Agent 不犯同样的错、越做越好。
- **能力生成回路**：当现有能力不足以应对任务时，Agent 自动生成和加载新能力。

**能力生成路径**：

```
工具层（组合现有工具）→ 配置层（调整策略参数）→ 插件层（生成新插件）→ Core 层（极少使用）
```

**关键原则**：

- 能的解决方案不动就不动，优先低成本
- 所有新能力必须经过契约校验 + 安全审查 + 自动化测试
- 高影响变更可设置人类审批节点
- 能力随使用增长而持续扩展

### 5.3 超级终端 —— 一个 APP 替代所有 APP

后端 Agent 能做什么，前端就自动长出对应的界面——通过 Schema 驱动渲染：后端声明能力的数据结构，前端自动映射成表单控件、工作区、Widget。用户不需要安装新 APP、不需要等前端开发，打开就用。

**核心体验**：

- **无所不能**：后端能力即前端功能，无需额外开发
- **开箱即用**：新增场景自动出现界面
- **千人千面**：同一个终端，不同用户看到不同界面
- **随心智适应**：手机简化、桌面展开，每个设备都是最佳体验

### 5.4 飞轮效应

三个维度形成正向循环：

```
全能闭环（覆盖更多场景）
    ↓ 发现新需求
自进化（生成新能力）
    ↓ 能力增强
超级终端（自动生成界面）
    ↓ 用户体验提升
用户使用更多场景
    ↓ 回到全能闭环...
```

**最终状态**：一个越用越强的 AI 系统，用户只需要在关键节点做决策，系统会自己变得越来越能干。

> **开放协议是飞轮的底座**：三维飞轮要真正转起来，前提是不被锁死在一个封闭系统里。AgentOS 全面拥抱开放协议——MCP、Function Calling、Anthropic Tool Use——让能力可互通、插件可流通、生态可共建。没有开放协议，闭环、进化、终端都只是孤岛上的自娱自乐。

> 愿景定义"去哪里"，具体的版本节奏与落地步骤见 [ROADMAP.md](../ROADMAP.md)。

## 六、产品形态

### 6.1 用户视角

```
用户打开 Agent OS
    │
    ├─ 选择一个"角色"（或使用默认助手）
    │   ├─ 专业助理 —— 生产力模式
    │   ├─ 亲密伙伴 —— 情感模式
    │   ├─ 创意搭档 —— 灵感模式
    │   └─ 自定义角色 —— 任何你想要的
    │
    ├─ 选择交互方式
    │   ├─ 对话 —— 聊天式交互
    │   ├─ 工作区 —— 看文档、看代码、看设计稿
    │   ├─ 悬浮窗 —— 在其他软件中随时召唤
    │   └─ 全屏 —— 沉浸式体验
    │
    └─ 开始做事
        ├─ "帮我写一部小说" → 创意生产闭环
```
