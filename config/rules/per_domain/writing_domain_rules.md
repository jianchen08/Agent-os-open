# 写作领域规则

---

## 一、系统提示词架构（三层）

借鉴 NovelAI 的"长期记忆 + 短期上下文 + 即时标签"分层模型，构建三层提示词架构。

### 1.1 长期设定层（Role-Play System）

贯穿整个项目的固定设定，相当于 NovelAI 的 Lorebook + 角色卡。

**角色卡标准结构**：

| 字段 | 说明 | 示例 |
|------|------|------|
| Name | 角色全名 | 林远舟 |
| Role | 角色定位 | 主角 / 反派 / 配角 |
| Age | 年龄 | 28 |
| Personality | 3-5个性格关键词 | 冷静、执拗、表面温和内心偏执 |
| Speech Pattern | 说话方式/口癖 | 短句为主，不用感叹号，紧张时反复说"没事" |
| Motivation | 核心动机 | 查清父亲死因 |
| Background | 背景经历 | 法医出身，转行做私家侦探 |
| Relationships | 人物关系 | 与女主苏晓是前同事关系 |
| Skills | 能力特长 | 微表情读取、格斗术 |
| Weaknesses | 弱点/缺陷 | 幽闭恐惧症、信任问题 |

**世界书标准结构**：

| 字段 | 说明 |
|------|------|
| World Name | 世界名称 |
| Genre | 体裁（玄幻/科幻/言情/悬疑等） |
| Time Period | 时代背景 |
| Geography | 地理设定 |
| Magic System / Tech | 魔法/科技体系 |
| Social Structure | 社会结构 |
| Key Rules | 硬规则（不可违反的世界法则） |
| Taboos | 禁忌（世界中的文化/社会禁忌） |
| Terminology | 术语表（专有名词及定义） |

### 1.2 短期引导层（Author's Note）

每章/每场景的即时指令，只影响当前段落。

- **本章目标**：本章要推进什么情节？
- **本场景情绪**：紧张/温馨/悲伤/兴奋？
- **需注意的伏笔**：本章需要回收或埋下哪些线索？
- **POV角色**：本章从谁的视角展开？
- **风格微调**：本章是否有特殊的文风要求？

### 1.3 即时标签层（Tags）

每段生成的即时控制参数。

| 标签 | 可选值 | 说明 |
|------|--------|------|
| genre | 玄幻/科幻/言情/悬疑/都市/历史 | 体裁 |
| style | 严肃/轻松/黑暗/幽默 | 整体文风 |
| pov | 第一人称/第三人称限知/全知 | 视角 |
| tense | 过去时/现在时 | 时态 |
| pace | 快/中/慢 | 节奏密度 |
| tone | 正剧/喜剧/悲剧 | 基调 |

## 二、情节结构模板（可切换）

根据项目需要选择合适的结构模板，不同模板可组合使用。

### 2.1 雪花法（严谨规划）

适合长篇连载，10步递进式生成：

```
1) 一句话概括 → 2) 一段话概括（5句对应5个情节点）→ 3) 角色简表 → 
4) 一页人物表 → 5) 详细大纲 → 6) 角色长表 → 7) 场景清单 → 
8) 场景详表 → 9) 场景描写 → 10) 撰写第一稿
```

### 2.2 三幕式（通用框架）

适合中短篇，25/50/25比例：

- **第一幕（25%）**：建置 → 触发事件 → 第一情节点
- **第二幕（50%）**：上升动作 → 中点 → 下降动作 → 第二情节点
- **第三幕（25%）**：高潮 → 下降 → 解决

### 2.3 英雄之旅（深度叙事）

适合奇幻/冒险/成长类，17阶段：

```
分离：平凡世界 → 冒险召唤 → 拒绝召唤 → 遇见导师 → 跨越门槛 → 试炼伙伴
启蒙：接近最深处 → 严酷考验 → 奖赏 → 回返之路
归来：复活 → 携药归来
```

### 2.4 Save the Cat（节奏控制）

适合精确控制节奏，15节拍：

```
Opening Image → Theme Stated → Setup → Catalyst → Debate →
Break Into Two → B Story → Fun and Games → Midpoint →
Bad Guys Close In → All Is Lost → Dark Night of the Soul →
Break Into Three → Finale → Final Image
```

### 2.5 七点结构（短篇/网文）

适合快速搭建大纲：

```
钩子 → 第一情节点 → 第一钉子 → 中点 → 第二钉子 → 第二情节点 → 解决
```

### 2.6 起承转合（中文传统）

适合中文写作的微观节拍，每个章节内部都应有一个小型的起承转合：

- **起**：事件起因，引入场景
- **承**：承接展开，发展情节
- **转**：转折引入，制造冲突
- **合**：归结收束，留下余韵

## 三、工作流节点

借鉴 Sudowrite 功能矩阵，映射为工作流节点。

### 3.1 Write（续写）

- 在光标位置续写后文
- 需要注入：前文上下文 + 角色卡 + 世界书 + 本章目标
- 输出要求：保持文风一致，推进情节

### 3.2 Rewrite（润色）

- 对选中句子/段落改写
- 改写维度：语句通顺、去除AI腔、增强画面感、调整节奏
- 保持原意不变，只改善表达

### 3.3 Describe（描写）

- 基于五感（视觉/听觉/嗅觉/味觉/触觉）和隐喻的感官细节生成
- 适用于：场景描写、人物外貌、动作细节、环境氛围
- 原则：Show Don't Tell

### 3.4 Brainstorm（头脑风暴）

- 按主题提供多个创意方向
- 每个方向附简要说明和可行性评估
- 不做评判，只做发散

### 3.5 Feedback（审稿）

- 读完全文，提供改进方向（见第四章审稿清单）
- 输出结构化审稿报告

### 3.6 Outline（大纲）

- 基于指定情节结构模板生成大纲
- 大纲粒度：卷 → 章 → 场景
- 每个节点标注：情节点类型、涉及角色、情感弧线

## 四、审稿清单（10维度）

章节完成后，按以下10个维度逐项检查。

### 4.1 情节结构

- [ ] 是否有清晰的结构（三幕/起承转合/七点）？
- [ ] 触发事件是否在前1/4出现？
- [ ] 中点是否有反转或重大揭示？
- [ ] 高潮是否在75%-90%位置？
- [ ] 伏笔是否埋下？是否回收？
- [ ] 是否有"流水账"或无目的场景？

### 4.2 人物一致性

- [ ] 角色行为是否符合角色卡设定？
- [ ] 性格是否前后一致（不"人设崩塌"）？
- [ ] 动机是否合理、可信？
- [ ] 说话方式是否符合角色口癖/句式？
- [ ] 配角是否工具化、扁平化？

### 4.3 视角人称

- [ ] 视角是否一致（无中途跳POV）？
- [ ] 第一人称是否仅展示主角所知？
- [ ] 全知视角是否解释了"读者何时知道"？
- [ ] 时态是否一致？

### 4.4 对话质量

- [ ] 每句对话是否推动情节或揭示角色？
- [ ] 是否能区分不同角色的说话方式？
- [ ] 是否避免"演讲式"独白？
- [ ] 对话标签是否多样化？

### 4.5 叙述描写

- [ ] 是否"展示而非告诉"（Show, Don't Tell）？
- [ ] 描写是否服务于情节/情绪？
- [ ] 五感描写是否均衡？
- [ ] 是否有大段无关的"信息倾倒"（Info Dump）？

### 4.6 对话vs叙述比例

| 体裁 | 对话占比 | 叙述占比 |
|------|----------|----------|
| 动作冒险 | 30% | 70% |
| 言情 | 50% | 50% |
| 文学 | 25% | 75% |
| 网文爽文 | 20-30% | 70-80% |

通用建议：每章对话不应超过50%。

### 4.7 节奏控制

- [ ] 场景长度是否合理？
- [ ] 是否有"张-弛"交替（呼吸感）？
- [ ] 章节末尾是否有钩子（Chapter Hook）？
- [ ] 是否有"该快不快、该慢不慢"问题？

### 4.8 世界观遵循度

- [ ] 魔法/科技体系规则是否前后一致？
- [ ] 术语使用是否统一？
- [ ] 地理/时间/社会规则是否被破坏？
- [ ] 文化/历史背景是否真实可信？

### 4.9 语言文风

- [ ] 句式是否有变化（长/短句交替）？
- [ ] 是否有重复用词、口头禅？
- [ ] 比喻/意象是否新鲜（非陈词滥调）？
- [ ] 是否符合体裁语域？

### 4.10 开头结尾

- [ ] 第一章前500字是否能吸引读者？
- [ ] 是否有"开场钩子"？
- [ ] 结尾是否解决核心冲突？
- [ ] 是否有余韵？
- [ ] 是否"草草收尾"或"戛然而止"？

## 五、体裁差异化配置

不同体裁应预置不同的规则配置，在角色卡/世界书创建阶段即确定。

### 5.1 网文爽文

- 节奏：快，每章必须有"爽点"
- 对话比例：20-30%
- 结构重点："黄金三章"（前三章必须抓住读者）
- 情绪密度：高，每章至少一个情绪高潮
- 特殊要求：章末钩子强制

### 5.2 严肃文学

- 节奏：慢，注重内心描写
- 对话比例：25%
- 结构重点：人物弧光 > 情节推进
- 文笔要求：考究、有个人风格
- 特殊要求：避免通俗化表达

### 5.3 言情

- 节奏：中等
- 对话比例：50%
- 结构重点：感情线推进（相遇→阻碍→升温→分离→重逢）
- POV：可多角色轮换
- 特殊要求：情感描写细腻

### 5.4 悬疑推理

- 节奏：渐进式加速
- 结构重点：伏笔密集，线索回收
- 特殊要求：误导与揭示的设计
- 审查重点：逻辑自洽、线索公平呈现

### 5.5 科幻/奇幻

- 节奏：视内容而定
- 结构重点：世界观构建
- 特殊要求：术语统一、规则自洽
- 审查重点：世界观遵循度优先

## 六、AI 写作缺陷检测规则

### 6.1 一致性缺陷（最高优先）

| 缺陷 | 表现 | 检测方法 |
|------|------|----------|
| 人设漂移 | 角色行为不符合早期设定 | 对比角色卡逐项检查 |
| 时间线混乱 | 事件顺序前后矛盾 | 绘制时间线图 |
| 空间错位 | 角色瞬移、缺交代 | 追踪角色位置变化 |
| 称呼混乱 | 角色间关系错乱 | 梳理关系图 |

### 6.2 文风缺陷

| 缺陷 | 表现 | 禁用词/模式 |
|------|------|-------------|
| AI腔 | 模板化句式 | "值得注意的是"、"总而言之"、"首先、其次"、"不可否认" |
| 过度修辞 | 每段都加比喻 | 同一段不超过2个比喻 |
| 情感夸张 | 滥用情绪描写 | "眼眶泛红"、"心跳加速"每章不超过1次 |
| 缺乏留白 | 把每件事都说透 | 允许读者自行推断的部分 |

### 6.3 情节缺陷

- **金手指过重**：主角无所不能 → 需设置合理限制
- **巧合过多**：解决问题靠"刚好"和"恰好" → 每卷不超过1次巧合解围
- **冲突人为**：为制造冲突而制造冲突 → 冲突须有因果链
- **伏笔遗失**：前面埋的线后面不收 → 建立伏笔追踪表

### 6.4 描写缺陷

- **环境单一**：反复同一类比喻 → 五感轮换
- **动作单薄**：只写"他打了她"不写怎么打 → 补充动作细节
- **对话单一**：所有角色用同一种口吻 → 差异化说话方式
- **心理泛滥**：大段内心独白替代行动 → 用行动展示心理

## 七、多轮审稿顺序

审稿不是一次性完成，而是按以下顺序多轮进行：

```
第一轮：一致性审稿 → 检查人设、时间线、世界观
第二轮：节奏审稿 → 检查张弛分布、钩子、比例
第三轮：文风审稿 → 检查AI腔、句式变化、用词
第四轮：结尾审稿 → 检查收束、余韵、情感落点
```

每轮审稿只关注该轮的重点，不混合检查，避免遗漏。
