# Environment Setup Agent 规则文件

本文档定义了 environment_setup_agent 在执行环境配置任务时必须遵守的规则。

---

## 依赖管理规范

### 1. Lockfile锁定

- **版本锁定** [error]: 所有依赖必须使用lockfile锁定精确版本，确保可重复构建
- **提交lockfile** [error]: lockfile必须纳入版本控制，禁止加入忽略列表
- **lockfile一致性** [error]: 安装依赖时必须同步更新lockfile，禁止手动修改

### 2. 版本固定

- **禁止浮动版本** [error]: 禁止使用浮动版本符（如^、~、*、latest），必须指定精确版本
- **版本范围限制** [warning]: 依赖版本范围应明确指定，避免自动升级引入不兼容变更
- **升级策略** [warning]: 依赖升级应单独操作，禁止与其他代码变更混合

### 3. 公私分离

- **生产与开发分离** [error]: 生产依赖与开发/测试依赖必须明确区分
- **开发依赖标识** [error]: 开发/测试依赖必须标记为开发依赖，不进入生产环境
- **可选依赖** [warning]: 可选依赖应明确标注并提供降级方案

### 4. 漏洞扫描

- **安全扫描** [error]: 每次安装/更新依赖必须执行安全漏洞扫描
- **高危漏洞** [error]: 发现高危漏洞必须立即修复或移除依赖
- **许可证合规** [warning]: 检查依赖的许可证是否与项目兼容
- **扫描报告** [warning]: 生成漏洞扫描报告并记录修复措施

---

## 环境标准化

### 5. 独立工作空间

- **项目隔离** [error]: 每个项目使用独立的工作空间/虚拟环境，禁止全局依赖污染
- **环境可复现** [error]: 环境配置必须可通过配置文件完全复现
- **环境文档** [warning]: 提供环境搭建的完整步骤文档

### 6. 工具链检测

- **完整性检查** [error]: 安装前必须检测工具链完整性（编译器、运行时、构建工具）
- **版本兼容性** [error]: 工具链版本必须与项目要求兼容
- **缺失提示** [warning]: 缺失工具时提供清晰的安装指引

### 7. 配置管理

- **配置文件化** [error]: 环境变量和配置必须通过配置文件管理，禁止硬编码
- **敏感信息** [error]: 密钥、令牌等敏感信息必须使用环境变量或密钥管理服务，禁止写入代码或配置文件
- **配置模板** [warning]: 提供配置文件模板，包含所有必需配置项的说明
- **环境区分** [warning]: 开发/测试/生产环境的配置应明确分离

### 8. 清理机制

- **依赖清理** [warning]: 提供依赖清理工具，移除未使用的依赖
- **缓存清理** [warning]: 提供缓存清理机制，避免构建缓存导致的问题
- **环境重置** [warning]: 提供环境重置脚本，可快速恢复到干净状态
