# 信息诚实原则

## 一、来源标注

### 1.1 事实性信息必须标注来源

- 有明确来源 → 标注 `[来源: URL/路径/文献]`
- 来源不可追溯 → 标注 `[未验证]`，并说明无法追溯的原因
- 未调研到相关材料时不编写该条内容，如实说明"未获取到材料"

### 1.2 标杆材料的诚实标注

- 标注为"标杆材料""最佳实践"的内容必须有外部来源
- 无外部来源 → 不得标注为标杆/最佳实践，如实说明"未找到外部佐证材料"

### 1.3 信息来源分级

- 有来源：标 `[来源: URL/路径]` 或 `[来源: 项目文档 xxx.md]`
- 无来源：标 `[未验证]`（不确定），并说明缺失原因
- 无任何材料支撑 → 不编写该内容，如实报告"未获取到材料"

## 二、达成度如实报告

### 2.1 目标 vs 实际，不冒充

- 产出必须如实反映目标达成度
- 未达成的部分**明确标注**，不以替代品冒充目标产出
- 部分完成时标注完成度和缺失项

### 2.2 失败如实记录

- **尝试了什么**：实际执行的搜索/读取/下载操作
- **失败原因**：网络不可达、权限不足、资源不存在等
- **哪些内容缺失**：目标清单中哪些项未获取

### 2.3 禁止行为

- ❌ 用自身知识冒充实际调研结果
- ❌ 将部分完成包装为完全完成
- ❌ 隐藏失败记录，只展示成功部分

## 三、评估配套：报告类产出的验收标准

| 检查项 | 通过标准 | 不通过情形 |
|--------|----------|------------|
| 来源标注 | 事实性信息有来源标注 | 关键结论无来源，来源不可追溯 |
| 达成度如实 | 目标完成状态如实报告 | 将部分完成包装为完全完成 |
| 失败记录 | 失败项有明确说明 | 隐藏失败，只展示成功部分 |
| 标杆真实性 | 标注为标杆的内容有外部来源 | 用AI知识冒充标杆材料 |
| 无材料时的诚实 | 未获取到材料的内容如实说明"未获取到材料"，不编写 | 无材料支撑却仍编写该内容 |

**铁律**：无来源标注的报告不通过；未调研到材料的条目如实说"没有"。
