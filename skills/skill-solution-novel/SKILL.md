# 小说方案规划

## 领域需求与流程调研

进入方案设计前，先调研小说项目的选题与创作需求。网文行业把动笔前的选题称为"选品"，是商业化成败前提（依据阅文/番茄创作实践）。

| 调研维度 | 要搞清的问题 | 调研方式 |
|----------|------------|----------|
| **选题与题材** | 原创/同人？题材类型（玄幻/都市/科幻/言情等）？该类型的代表作与套路框架是什么？[依据: 知乎网文流程/阅文创作学堂] | 与用户澄清 + 派发 research_agent 调研同题材头部作品 |
| **平台风格要求** | 目标平台（起点/晋江/番茄）？平台风格是强约束：番茄重快节奏爽文+黄金三章+爽点密集；起点容忍慢热铺垫、世界观更厚；晋江重人设与情感线。[依据: 上观新闻/什么值得买平台对比] | 派发 research_agent 调研平台规则 + 用户确认 |
| **目标读者画像** | 读者年龄/性别/阅读偏好？Z 世代为核心群体的趋势？[依据: 中国社科网 2025 报告] | 派发 research_agent + 用户确认 |
| **核心冲突与前提** | 故事的一句话前提是什么（Egri: 每个故事证明一个真理）？核心冲突是什么？主角的欲望与障碍？[依据: Egri《编剧的艺术》] | 与用户深入讨论确认 |
| **力量体系** | 是否有力量体系（修仙/魔法/科技）？体系必须设天花板和代价，否则升级贬值。[依据: 中国作家网/爱潜水的乌贼专访] | 与用户讨论 |
| **商业定位** | 付费阅读/免费广告分成/IP 改编？对标作品与差异化卖点？[依据: 阅文财报/大阅文战略] | 派发 research_agent + 用户确认 |
| **执行节奏** | 日更/周更/完结再发？风格打样需求？是否需要逐章审稿？ | 与用户确认，影响任务链编排 |

**调研产出**：调研报告引用写入方案总纲；稳定设定写入 `.project/`（世界观+力量体系→`worldbuilding.md`，平台风格要求→`style_guide.md`）。

## 方案内容框架

### 一、世界观体系（世界设定）

世界观是小说世界的基石，方案中必须明确以下维度：

| 维度 | 说明 | 示例 |
|------|------|------|
| **时空背景** | 故事发生的时代、地域、地理环境 | 架空大唐、星际殖民时代 |
| **规则体系** | 世界的运行法则（物理/魔法/科技/社会规则） | 魔法等级体系、灵力修炼体系 |
| **势力格局** | 主要势力、阵营划分、力量对比 | 宗门、帝国、反抗军 |
| **文明水平** | 科技/魔法/文化发展程度 | 蒸汽朋克、修真文明 |
| **独特设定** | 区别于其他世界的特色设定 | 独特的力量来源、种族设定 |

**世界设定原则**：
1. **一致性**：所有设定在逻辑上自洽，不前后矛盾
2. **合理限制**：规则有明确的限制条件，避免"无所不能"
3. **故事驱动**：设定服务于故事，不为设定而设定
4. **留白空间**：给读者留下想象和探索的空间

### 二、人物体系（人设设计）

人物是故事的核心，方案中为每个主要角色定义：

| 维度 | 说明 |
|------|------|
| **基本信息** | 姓名、年龄、性别、身份、外貌特征 |
| **性格画像** | 核心性格 traits（3-5个关键词）、性格形成原因 |
| **能力体系** | 特殊能力/技能/知识背景，能力成长路径 |
| **动机目标** | 核心驱动力（想达成什么）、行为动机、内心冲突 |
| **关系网络** | 与其他角色的关系、阵营归属、感情线 |
| **人物弧光** | 故事开始→结束的性格/认知变化轨迹 |
| **标志特征** | 口头禅、习惯动作、标志性物品等区隔元素 |

**人设设计原则**：
1. **区隔性**：每个主要角色有独特的辨识度
2. **动态性**：角色在故事中有成长和变化
3. **驱动性**：角色的动机推动剧情发展
4. **代入感**：读者能共情角色的处境和选择

### 三、大纲体系（故事框架）

故事的整体骨架，方案中包含：

| 层级 | 内容 | 详细程度 |
|------|------|----------|
| **故事梗概** | 一句话概括核心冲突 | 一句话 |
| **内容简介** | 300-500字的完整故事简介 | 小段落 |
| **三幕/起承转合** | 主要情节阶段的划分和转折点 | 每个阶段2-3句话 |
| **情节点** | 核心情节点/爆点/反转节点 | 列表 |
| **节奏设计** | 各阶段的情绪曲线和节奏分布 | 表格或图示 |

**大纲结构参考**（适合网文的节奏节奏）：
```
第一幕（开头1/4）：引入世界观→主角亮相→核心冲突（钩子）→第一转折点
第二幕（中间1/2）：升级挑战→遇挫成长→新势力介入→危机升级→低谷/转折
第三幕（结尾1/4）：最终准备→高潮对决→结局收束→余韵/伏笔
```

### 四、章节体系（执行单元）

方案中按照章节/卷来组织写作任务：

| 维度 | 说明 |
|------|------|
| **卷/部划分** | 按故事阶段划分卷，每卷有独立的起承转合 |
| **章节粒度** | 每章字数范围（网文通常2000-3000字/章） |
| **章节摘要** | 每章核心内容一句话概括 |
| **钩子设计** | 章末钩子（悬念/反转/新信息），维持读者追读 |
| **多线并行** | 多视角/多线叙事时，标注每章主线 |

### 五、作品定位（商业/艺术考量）

| 维度 | 说明 |
|------|------|
| **目标读者** | 读者画像、阅读习惯、偏好类型 |
| **平台适配** | 适合的平台（起点/晋江/番茄等）及其风格要求 |
| **商业化潜力** | IP改编潜力、周边开发方向 |
| **同类对标** | 参考同类作品、差异化定位 |

## 产出物拆分约定

小说类方案的产出**拆成三类**，各自职责不重叠。这是方案规划阶段必须遵守的产出模型。

### A. `.project/` 文档（稳定真相源，方案阶段定稿）

世界观、人设、大纲等稳定设定写入 `.project/`，**平铺不分子目录**，遵守 `document_context_rules.md`。
**禁止写入**具体章节正文、对白原稿、逐章分镜。

| 文件 | 装什么 |
|------|--------|
| `worldbuilding.md` | 时空背景、规则体系、势力格局、文明水平、独特设定与限制条件 |
| `character_bible.md` | 主要角色人设卡：基本信息、性格画像、能力体系、动机目标、关系网络、人物弧光、标志特征 |
| `plot_outline.md` | 故事梗概、三幕/起承转合结构、情节点清单、节奏曲线、伏笔与回收表 |
| `style_guide.md` | 文风约定（叙事视角、语言风格、用词偏好）、叙事节奏与钩子规则、平台适配要求 |

> 不是每个文件都必须有：方案涉及哪类信息就写哪类。无新内容的文件不创建。

### B. `docs/{title}_solution.md`（方案总纲，项目导航层）

总纲是项目导航与索引，服务于 `.project/` 文档和任务链。**只装导航层**：
- 作品定位与目标（题材类型/目标读者/目标平台/对标作品）
- 调研报告引用（指向 `docs/working/{title}_research_report.md` 路径，**不摘抄调研内容**）
- **讨论要点与关键创作决策**（讨论中确定的关键点、设定/方向的选择理由——为什么这么定，不是设定本身）
- **`.project/` 文件概览**（列出方案定稿了哪些 `.project/` 文件，每个一句话定位，让人知道去哪查世界观/人设/大纲等细节）
- 方案级 AC 总表（机读 YAML，侧重故事完整性与读者体验）、变更记录、任务列表

世界观细节、人设卡、大纲、章节规划、调研内容摘抄**禁止放在这里**，它们下沉到 `.project/`、调研报告或任务文件。

### C. `docs/tasks/task_XX_*.md`（自包含任务文件）

任务文件必须自带「执行方案与设计」节，写手拿到即可开工。小说领域该节写**创作契约级**内容：

| 必写项 | 说明 |
|--------|------|
| 本章核心内容点 | 本章必须推进的剧情、完成的事件（引用 `.project/plot_outline.md` 情节点） |
| 涉及角色与关系 | 本章出场角色及其当前状态（引用 `.project/character_bible.md`） |
| 钩子设计 | 章末悬念/反转/新信息，承接下一章的方式 |
| 文风约束 | 本章适用的叙事约束（引用 `.project/style_guide.md`，如视角切换、节奏要求） |

**颗粒度**：到"本章写什么、用什么设定、怎么留钩子"，不到具体对白和分镜。让写手知道"写什么"，而非"怎么写每一句"。

## 注意事项

- 人设和世界观需要留出调整空间，不要在方案阶段锁定所有细节
- 创作类方案强调"方向性指导"而非"刚性规格"
- 验收标准侧重故事完整性和读者体验，而非字数达标
- **力量体系必须有天花板和代价**，否则升级贬值、爽点失效 [依据: 中国作家网/爱潜水的乌贼专访]
- **主线不超过两条**，感情线不作主线，否则剧情崩坏、人设崩塌 [依据: 阅文创作学堂/不祈十弦]
- 人设用 Egri 三维（生理/社会/心理）避免脸谱化，角色行为须与其三维和动机一致 [依据: Egri《编剧的艺术》]
